# Brainfuck Compiler

Компилятор языка [Brainfuck](https://ru.wikipedia.org/wiki/Brainfuck) в нативный код для Linux x86-64, написанный на Go.

## Описание

Данный проект транслирует исходный код на Brainfuck напрямую в ассемблер (`nasm`), после чего собирает и компонует его в исполняемый ELF-файл. Компиляция происходит в три этапа:

1. **Токенизация** — исходный код разбивается на токены.
2. **Генерация кода** — по токенам генерируется ассемблерный код (`out.s`).
3. **Сборка и компоновка** — `nasm` собирает `.s` в объектный файл, `ld` компонует его со стандартной библиотекой (`bfstd.a`) в исполняемый файл.

## Поддерживаемые команды Brainfuck

| Команда | Описание |
|---------|----------|
| `>`     | Переместить указатель данных вправо |
| `<`     | Переместить указатель данных влево |
| `+`     | Увеличить значение текущей ячейки на 1 |
| `-`     | Уменьшить значение текущей ячейки на 1 |
| `.`     | Вывести значение текущей ячейки как ASCII-символ |
| `D`     | **Расширение:** вывести значение текущей ячейки как число (отладка) |
| `[`     | Начало цикла: если текущая ячейка равна 0, перейти к `]` |
| `]`     | Конец цикла: вернуться к соответствующей `[` |

> **Примечание:** стандартная команда `,` (ввод) пока что не реализована.

## Использование

```
brainfuck [--оptions] <input> [-o output]
```

### Аргументы

| Аргумент | Описание |
|----------|----------|
| `<input>` | Путь к исходному файлу Brainfuck (обязательный) |
| `-o <file>` | Имя выходного исполняемого файла (по умолчанию: `a.out`) |
| `-S` | Только компиляция (генерация `.s`-файла, без ассемблирования и компоновки) |
| `-c` | Компиляция и ассемблирование (до `.o`-файла, без компоновки) |

### Примеры

```bash
# Скомпилировать и запустить
brainfuck hello.bf -o hello
./hello

# Только сгенерировать ассемблерный код
brainfuck hello.bf -S

# Скомпилировать до объектного файла
brainfuck hello.bf -c
```

## Структура проекта

| Файл | Назначение |
|------|------------|
| [`main.go`](main.go) | Точка входа: чтение файла, вызов токенизатора и кодогенератора, запуск ассемблера и компоновщика |

| [`codegen.go`](codegen.go) | Генератор кода: превращает токены в ассемблерный код NASM |
| [`bfstd.a`](bfstd.a) | Статическая библиотека с рантайм-функциями (работа с памятью, ввод/вывод) |

## Статическая библиотека (`bfstd.a`)

Пока что проект использует предварительно собранную статическую библиотеку `bfstd.a`, которая предоставляет следующие рантайм-функции:

- `vector_init` — инициализация вектора (ленты памяти)
- `read_cell` — чтение значения текущей ячейки
- `next_cell` / `prev_cell` — перемещение по ленте
- `add_cell` / `sub_cell` — инкремент/декремент ячейки
- `print_cell` — вывод ячейки как ASCII-символ
- `print_cell_num` — вывод ячейки как числа (отладка)
