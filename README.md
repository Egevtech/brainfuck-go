# Brainfuck Compiler

Компилятор языка [Brainfuck](https://ru.wikipedia.org/wiki/Brainfuck) в нативный код для Linux **x86-64** и **ARM64 (AArch64)**, написанный на Go.

## Описание

Данный проект транслирует исходный код на Brainfuck напрямую в ассемблер, после чего собирает и компонует его в исполняемый ELF-файл. Компиляция происходит в три этапа:

1. **Токенизация** — исходный код разбивается на токены.
2. **Генерация кода** — по токенам генерируется ассемблерный код (`out.s`). Целевая архитектура определяется автоматически по архитектуре хоста (`runtime.GOARCH`).
3. **Сборка и компоновка** — ассемблер собирает `.s` в объектный файл, `ld` компонует его со стандартной библиотекой (`bfstd-<arch>.a`) в исполняемый файл.

## Поддерживаемые команды Brainfuck

| Команда | Описание |
|---------|----------|
| `>`     | Переместить указатель данных вправо |
| `<`     | Переместить указатель данных влево |
| `+`     | Увеличить значение текущей ячейки на 1 |
| `-`     | Уменьшить значение текущей ячейки на 1 |
| `.`     | Вывести значение текущей ячейки как ASCII-символ |
| `D`     | **Расширение:** вывести значение текущей ячейки как число (отладка) |
| `[`     | Начало цикла: если текущая ячейка равна 0, перейти к `]` |
| `]`     | Конец цикла: вернуться к соответствующей `[` |

> **Примечание:** стандартная команда `,` (ввод) пока что не реализована.

## Использование

```
brainfuck [--оptions] <input> [-o output]
```

### Аргументы

| Аргумент | Описание |
|----------|----------|
| `<input>` | Путь к исходному файлу Brainfuck (обязательный) |
| `-o <file>` | Имя выходного исполняемого файла (по умолчанию: `a.out`) |
| `-S` | Только компиляция (генерация `.s`-файла, без ассемблирования и компоновки) |
| `-c` | Компиляция и ассемблирование (до `.o`-файла, без компоновки) |

### Примеры

```bash
# Скомпилировать и запустить
brainfuck hello.bf -o hello
./hello

# Только сгенерировать ассемблерный код
brainfuck hello.bf -S

# Скомпилировать до объектного файла
brainfuck hello.bf -c
```

## Зависимости
### Сборки
- [Go](https://go.dev/) 1.26+
- **x86-64:** [NASM](https://www.nasm.us/) — ассемблер для x86-64
- **ARM64:** [GAS](https://sourceware.org/binutils/docs/as/) (GNU Assembler, `as`) — входит в состав binutils
- [ld](https://sourceware.org/binutils/docs/ld/) — компоновщик GNU
### Библиотеки
- [go-arg](https://github.com/alexflint/go-arg) — парсинг аргументов командной строки

## Сборка

```bash
go build -o brainfuck
```

## Статические библиотеки (`bfstd-x86_64.a` / `bfstd-arm64.a`)

Проект использует предварительно собранные статические библиотеки под целевую архитектуру. Выбор библиотеки происходит автоматически на этапе компоновки в зависимости от архитектуры хоста. Библиотеки предоставляют следующие рантайм-функции:

- `vector_init` — инициализация вектора (ленты памяти)
- `read_cell` — чтение значения текущей ячейки
- `next_cell` / `prev_cell` — перемещение по ленте
- `add_cell` / `sub_cell` — инкремент/декремент ячейки
- `print_cell` — вывод ячейки как ASCII-символ
- `print_cell_num` — вывод ячейки как числа (отладка)

## Поддерживаемые архитектуры

| Архитектура | Ассемблер | Статическая библиотека | Динамический компоновщик |
|-------------|-----------|----------------------|--------------------------|
| x86-64 (amd64) | NASM (`nasm -felf64`) | `bfstd-x86_64.a` | `/lib64/ld-linux-x86-64.so.2` |
| ARM64 (AArch64) | GAS (`as`) | `bfstd-arm64.a` | `/lib/ld-linux-aarch64.so.1` |

Архитектура определяется автоматически во время компиляции через `runtime.GOARCH` — никаких дополнительных флагов или переменных окружения не требуется.
